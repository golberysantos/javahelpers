# 📜 Manifesto do JavaHelper AI


## Nossa missão

Construir software compreensível, evolutivo e testável.

## Nossa visão

Demonstrar que arquitetura de software é uma ferramenta para facilitar mudanças e não apenas organizar código.

## Nossos valores

- Clareza acima de complexidade.
- Testes fazem parte da implementação.
- O domínio vem antes da tecnologia.
- Código é um meio.
- Arquitetura é uma forma de pensar.

1. O domínio vem antes da tecnologia.

Nunca vamos criar uma classe porque o Spring Boot permite.

Vamos criá-la porque ela resolve um problema do domínio.

2. Clareza é mais importante que inteligência.

Prefiro um código que qualquer desenvolvedor entenda em 30 segundos do que um código "genial" que só o autor compreende.

3. Testes fazem parte da implementação.

Um recurso não termina quando compila.

Ele termina quando está testado.

4. A arquitetura deve facilitar mudanças.

Toda decisão deve responder:

"Se daqui a um ano precisarmos mudar isso, será fácil?"

5. O projeto deve ensinar.

Cada classe deve ser capaz de ensinar alguma boa prática.
