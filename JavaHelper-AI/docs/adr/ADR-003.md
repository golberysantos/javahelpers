#ADR-003


#Uso de Mock como implementação inicial do mecanismo de conhecimento.

Status
 - Aceita

## Contexto

O projeto JavaHelper AI pretende utilizar modelos de IA para responder perguntas sobre Java.

Durante o desenvolvimento inicial, a utilização da API da OpenAI implicaria dependência de um serviço pago, dificultando testes automatizados, desenvolvimento offline e colaboração de outros desenvolvedores.

## Decisão

A primeira versão utilizará uma implementação Mock baseada em arquivo JSON como fonte de conhecimento.

Toda a comunicação com o mecanismo de conhecimento ocorrerá através de abstrações (interfaces), permitindo substituir futuramente a implementação Mock por uma integração com OpenAI, Spring AI ou outro mecanismo equivalente sem alterar as camadas superiores da aplicação.

Consequências

Positivas
Desenvolvimento independente de custos.
Testes determinísticos.
Facilidade para contribuir com o projeto.
Arquitetura desacoplada.
Evolução gradual.
Negativas
Respostas limitadas.
Busca baseada em palavras-chave.
Sem entendimento semântico.
