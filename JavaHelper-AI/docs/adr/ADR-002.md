ADR-002 – Estratégia Inicial de Organização de Pacotes, registrando:

Alternativas consideradas: Package by Layer e Package by Feature.
Decisão atual: iniciar com Package by Layer.
Critério para evolução: migrar para Package by Feature quando o domínio justificar.

Assim, quando chegarmos à Sprint dessa migração, não parecerá uma mudança de ideia. Ficará claro que ela fazia parte da evolução planejada da arquitetura desde o início.

Na minha opinião, essa é uma das maiores lições que podemos construir no JavaHelper AI: boas arquiteturas não são estáticas; elas evoluem de forma consciente, registrada e orientada pelo contexto.
